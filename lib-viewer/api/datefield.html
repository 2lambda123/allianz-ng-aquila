Directives

NxDatefieldDirective

Directive used to provide date processing functionality to an input.

Selector:

input[nxDatefield]

Exported as: nxDatefield
Properties
Name Description
@Input('nxDatefieldFilter')

datefieldFilter: (date: D) => boolean

Function that can be used to filter out dates within the datepicker and invalidate values in the datefield.
@Input('nxDatepicker')

datepicker: NxDatepickerComponent

The datepicker that this input is associated with.
@Input()

disabled: boolean

Whether the datepicker-input is disabled.
@Input('nxDisplayFormat')

displayFormat: string

Override the display format given with display.dateInput with the token NX_DATE_FORMATS
@Input('nxMax')

max: D | null

Sets the maximum valid date.
@Input('nxMin')

min: D | null

Sets the minimum valid date.
@Input('nxParseFormat')

parseFormat: string | string[]

Override the parse format given with parse.dateInput with the token NX_DATE_FORMATS.

The date value is not updated on a parseFormat change.
@Input()

readonly: boolean

Whether the datefield is readonly.
@Input('nxStrict')

strict: any

If supported by the date implementation enable strict parsing (applies to Moment's parse function here).

The date value is not updated on a strict change.
@Input()

value: D | null

Provide or read the current date. It's type depends on the chosen date implementation
@Output()

dateChange: EventEmitter>

Emits when a change event is fired on this <input>.
@Output()

dateInput: EventEmitter>

Emits when an input event is fired on this <input>.
Methods
getValidators(): ValidatorFn[]

Returns the validators of the datefield.

Deprecated
format()
Deletion target: 10.0.0 Calling `format` manually is not needed anymore and can be removed. This will force the current value to be parsed again by the given DateAdapter. That's a convenience method so you can trigger it manually.

NxDateValidators

Provides a set of date validators that can be used by form controls.

NxDatepickerInputEvent

An event used for datepicker input and change events. We don't always have access to a native input or change event because the event may have been triggered by the user clicking on the calendar popup. For consistency, we always use NxDatepickerInputEvent instead.

Properties
Name Description

target: NxDatefieldDirective

Reference to the datepicker input component that emitted the event.

targetElement: HTMLElement

Reference to the native input element associated with the datepicker input.

value: D | null

The new value for the target datepicker input.
